# Parallel Wave 并行波次指南

Parallel Wave 用于让多个执行会话同时处理多个互不冲突的任务。Wave 是调度方式，不改变任务责任边界。

TASK 是最小责任单位；Wave 不能替代单个 TASK 的记录、审查和验收。

## 核心概念

### Single Task

一个执行会话只执行一个任务。

适合：

- C/D 任务。
- 高风险任务。
- 任务边界不清的任务。
- 涉及核心模块、真实环境、实机测试、回归验收或用户决策的任务。

### Batch

一个执行会话顺序执行多个 A/B 小任务。

适合：

- 多个低风险文档任务。
- 多个低风险模板任务。
- 少量互不冲突的小代码任务。

### Parallel Wave

多个执行会话同时执行多个互不冲突的任务。

适合：

- 文件范围明确。
- 模块边界清楚。
- 任务之间无依赖。
- diff 可拆分。
- 可以分别审查和回滚的任务。

## 工作区隔离规则

- 任何进入 Parallel Wave 的代码任务，默认必须使用独立分支或独立 Worktree。
- 多个代码执行会话不得共享同一个工作区直接改代码。
- A 级纯文档任务可以共享主项目工作区。
- B/C 代码任务如果并行，必须有独立分支或 Worktree，并记录各自 base commit、HEAD 和 diff 范围。
- 如果用户明确要求多个代码任务共享同一个工作区，agent 必须标记高风险，并要求用户确认。
- 如果无法确认工作区隔离，不得启动代码并行。

## 可以进入 Parallel Wave 的任务

### A 级文档任务

可以并行。

条件：

- 文件范围明确。
- 不同时修改同一个核心文档。
- 不影响业务代码。
- 不需要实机测试。

### B 级小代码任务

可以谨慎并行。

条件：

- 修改文件不重叠。
- 不涉及核心模块。
- 不涉及构建、发布、依赖。
- 不涉及通信协议。
- 不涉及数据写入、删除、迁移。
- 使用独立分支或独立 Worktree。
- 可独立验证。
- 可独立回滚。

### C 级中等任务

默认不推荐并行，但允许和 A 级文档任务并行。

如需与其他代码任务并行，必须满足：

- 独立分支或独立 Worktree。
- 模块不重叠。
- 文件不重叠。
- 依赖关系清楚。
- 用户明确确认。

### D 级高风险任务

默认不允许进入代码并行。

D 级任务只能和只读分析、文档整理、审查任务并行，不得和其他代码任务同时执行。

## 禁止并行

以下情况必须串行：

- 两个任务修改同一文件。
- 两个任务修改同一核心模块。
- 两个任务修改同一接口、协议、DTO、schema。
- 一个任务依赖另一个任务结果。
- 任务涉及架构迁移。
- 任务涉及技术栈迁移。
- 任务涉及构建 / 发布 / 依赖。
- 任务涉及核心服务。
- 任务涉及进程间通信 / 通信协议。
- 任务涉及项目入口或 UI host 启动链路。
- 任务涉及前端全局状态流。
- 任务涉及数据写入、删除、迁移。
- 任务涉及真实软件插件或真实环境。
- 任务需要 UA5 / UA6 / UA7。
- 当前工作区已有来源不明的未提交改动。
- 并行代码任务无法确认独立分支或 Worktree。
- 多个代码执行会话共享同一工作区且未获得用户明确确认。
- agent 无法判断冲突风险。

## 文件锁 / 模块锁

每个并行任务开始前必须声明：

- 预计修改文件。
- 预计读取文件。
- 预计影响模块。
- 禁止修改文件。
- 禁止修改模块。

如果两个任务的“预计修改文件”重叠，不得并行。

如果两个任务的“预计影响模块”重叠，默认不得并行，除非用户明确确认。

核心文件默认需要独占锁，例如：

- 项目入口文件。
- 构建配置。
- 全局状态文件。
- 协议 / DTO / schema。
- 核心服务。
- 主流程编排文件。

## Parallel Wave 流程

1. 总览会话从 `TASK_BOARD` 中挑选候选任务。
2. 判断每个任务 A/B/C/D 等级。
3. 排除 D 级任务和高风险代码任务。
4. 检查任务依赖。
5. 检查文件锁 / 模块锁。
6. 检查 UA 等级，UA5 / UA6 / UA7 默认不得进入代码并行。
7. 检查代码任务是否具备独立分支或 Worktree。
8. 如果无法确认工作区隔离，不得启动代码并行。
9. 创建或更新 `WAVE-xxx.md`。
10. 为每个并行任务指定执行会话。
11. 每个执行会话只执行自己负责的任务。
12. 每个任务记录自己的 base commit、HEAD、diff 范围、验证结果和交接包。
13. 执行完成后交给 Review Hub。
14. Review Hub 可以按 Wave 批量审查，但必须逐任务输出结论。
15. 如果某个任务出现 P0/P1、范围越界或冲突，单独拉出修复，不影响其他已通过任务。
16. 所有任务按各自 UA 等级进入验收建议。
17. 用户确认后再合并或关闭。

## 默认并行规模

建议默认：

- A 级文档任务：最多同时 5 个。
- B 级小代码任务：最多同时 2 个。
- C 级任务：最多 1 个代码任务，可并行 1~2 个 A 级文档任务。
- D 级任务：不和其他代码任务并行。
- 总执行会话数量建议不超过 3 个。

如果用户明确提高并行度，agent 仍必须先做冲突检查。

## Review Hub 审查要求

- 可以集中审查一个 Wave。
- 必须逐任务审查，不得只输出 Wave 总结论。
- 每个任务输出 P0/P1/P2/P3、UA 等级和是否允许进入验收建议。
- 每个任务结论写回对应 `TASK` 文件。
- 发现冲突、P0/P1 或 C/D 风险时，将该任务移出 Wave 单独处理。
