# 项目语境（project_context）指南

项目语境（`project_context`）用于沉淀长期稳定、可复用的项目知识，例如领域词汇、核心流程、验证经验和常见坑。推荐在具体项目中维护 `docs/memory/PROJECT_CONTEXT.md`。

## 与其他文档的关系

- `PROJECT_INDEX.md`：项目入口、目录、运行和验证方式。
- `PROJECT_CONTEXT.md`：稳定语境、术语、业务约束、常见误解和验证经验。
- `DECISIONS.md` 或 ADR：已经确认的重要决策、原因和影响。
- TASK 文件：单个任务的事实来源，不替代长期语境。

## 可以记录

- 项目内稳定术语和缩写。
- 核心用户流程和关键约束。
- 常用验证方式、样例数据说明和测试入口。
- 已确认的架构边界、模块职责和禁止误改点。
- 多次出现的问题、排查路径和修复经验。

## 不得记录

- 聊天全文。
- 密钥、Token、账号密码、证书或私有配置。
- 本机绝对路径、私有账号、真实客户数据或个人隐私。
- 未经确认的猜测。
- 只对一次任务有效的临时状态。
